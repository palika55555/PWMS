# Oprava 404 chyby na Vercel

## Problém
Vercel vracia 404: NOT_FOUND

## Riešenie

### Krok 1: Skontrolujte Vercel Dashboard

1. Prejdite do **Settings** → **Build and Deployment**
2. Skontrolujte:
   - **Output Directory**: Musí byť `build/web` (s Override zapnutým)
   - **Build Command**: Musí byť **prázdne** (s Override zapnutým)
   - **Install Command**: Môže byť vypnuté

### Krok 2: Skontrolujte, či je build/web v Git

```bash
git ls-files build/web/index.html
```

Ak nič nevrátí, build/web nie je commitnutý.

### Krok 3: Vytvorte nový deployment

1. V Vercel Dashboard → **Deployments**
2. Kliknite na **Redeploy** na poslednom deploymente
3. Alebo vytvorte nový deployment manuálne

### Krok 4: Alternatíva - Nasadenie cez Vercel CLI

```bash
cd build/web
vercel --prod
```

Toto nasadí priamo z lokálneho build/web priečinka.

## Poznámka

Ak stále vidíte 404, skontrolujte:
- Či je `build/web/index.html` commitnutý v Git
- Či je Output Directory správne nastavený na `build/web`
- Či Vercel správne našiel build/web priečinok po clone z Git
